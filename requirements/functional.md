# Functional Requirements 

The functional requirements describe the specific services and functions that the **Cakelt** system must provide to satisfy the user needs.

| ID | Function | Description |
|:---|:---|:---|
| **FR1** | **Account Creation** | The system allows new users to register by providing email, name, and password. |
| **FR2** | **Secure Login** | Registered users can securely access their accounts. |
| **FR3** | **Cake Customization** | Users can select size, shape, flavors, and fillings for their custom cake. |
| **FR4** | **Decoration Tools** | Users can add custom text, colors, and specific toppings to the design. |
| **FR5** | **Visual Preview** | The system generates a visual representation of the customized cake before ordering. |
| **FR6** | **Shopping Cart** | Users can add standard or custom cakes to a cart for checkout. |
| **FR7** | **Order Tracking** | Users can view the live status of their order (Processing, Baking, Out for Delivery). |
| **FR8** | **Payment Processing** | The system integrates with payment gateways to handle credit card or Mada transactions. |
