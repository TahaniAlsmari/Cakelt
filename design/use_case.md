# Use Case Modeling

##  System Boundary and Interactions
The Use Case Diagram defines the interactions between our users (Customer and Admin) and the **Cakelt** system. It highlights the functional scope of the application.

![Use Case Diagram](../assets/use-case.png)

##  Actors Definition
* **Customer:** The primary actor who browses the menu, customizes cakes, and manages orders.
* **Payment Gateway:** An external actor that handles secure transaction processing.
* **Admin:** Manages the store inventory and order fulfillment (Internal actor).

##  Key Use Cases
* **Customize Cake:** The core feature allowing flavor and decoration selection.
* **Place Order:** The process of finalizing the selection and initiating payment.
